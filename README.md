# Studio54

Music & Audiobook management system. Lidarr-style acquisition workflow with MusicBrainz integration, SABnzbd automation, and a full audiobook library manager.

## Features

- **Music Library** — Artist/album management, MusicBrainz metadata, cover art, track ratings
- **Audiobook Library** — Author/series/book management with cover art and monitoring
- **Acquisition** — SABnzbd integration for automated NZB downloads
- **File Organization** — MBID-based file naming and directory structure
- **Background Tasks** — Celery workers for import, sync, fingerprinting, and organization
- **Audio Fingerprinting** — AcoustID integration for untagged file identification
- **Now Playing** — Real-time playback tracking
- **Log Viewer** — Integrated Dozzle instance filtered to Studio54 containers
- **Auto-Updates** — Optional Watchtower integration

## Quick Start

```bash
# Clone the repository
git clone https://github.com/tesimmons/Studio54.git
cd Studio54

# Run the installer (handles Docker, .env, directories, systemd)
./scripts/install.sh

# Or set up manually:
./studio54 setup
./studio54 start
```

After startup:
- **Web UI**: http://localhost:8009
- **API Docs**: http://localhost:8010/docs
- **Logs (Dozzle)**: http://localhost:9998

## Requirements

- Ubuntu 22.04 / 24.04 (other Linux distros may work)
- Docker and Docker Compose (installed by `scripts/install.sh`)
- 10 GB+ free disk space
- NVIDIA GPU optional (used by Ollama for AI metadata matching)

## Management CLI

All operations go through the `studio54` script:

```bash
./studio54 check          # Verify prerequisites
./studio54 setup          # First-run setup (create .env, directories)
./studio54 start          # Start all services
./studio54 stop           # Stop all services
./studio54 restart        # Restart all services
./studio54 status         # Show status, URLs, resource usage
./studio54 logs           # Follow all logs
./studio54 logs studio54-service   # Follow a specific service's logs

./studio54 deploy studio54-service   # Rebuild + redeploy (--no-cache)
./studio54 deploy studio54-web

./studio54 update         # Pull latest images and restart
./studio54 backup all     # Backup database, redis, cover art
./studio54 restore <file> # Restore from backup

./studio54 watchtower enable    # Start auto-update service
./studio54 watchtower disable   # Stop auto-update service
./studio54 watchtower status    # Show update status

./studio54 help           # Full command reference
```

## Configuration

Copy `.env.example` to `.env` and configure:

```bash
cp .env.example .env
nano .env
```

**Required values:**
- `STUDIO54_DB_PASSWORD` — auto-generated by `./studio54 setup`
- `STUDIO54_ENCRYPTION_KEY` — auto-generated by `./studio54 setup`
- `MUSIC_LIBRARY_PATH` — path to your music collection on the host
- `AUDIOBOOKS_PATH` — path to your audiobook collection on the host

**Optional API keys** (improves metadata quality):
- `FANART_API_KEY` — artist and album images from fanart.tv
- `LASTFM_API_KEY` — Last.fm scrobbling and metadata
- `ACOUSTID_API_KEY` — audio fingerprinting
- `SABNZBD_API_KEY` — automated NZB downloads
- `OLLAMA_URL` — AI-assisted metadata matching

## Deployment

After editing backend code, always use the deploy script to avoid Docker cache issues:

```bash
./studio54 deploy studio54-service   # rebuilds service + worker + beat
./studio54 deploy studio54-web       # rebuilds frontend
```

Manual deployment (if needed):
```bash
docker compose build --no-cache studio54-service
docker stop studio54-service studio54-worker studio54-beat
docker rm studio54-service studio54-worker studio54-beat
docker compose up -d studio54-service studio54-worker studio54-beat
docker image prune -f
```

## Auto-Start on Boot (systemd)

The installer configures Studio54 as a systemd service automatically. To manage it:

```bash
sudo systemctl status studio54
sudo systemctl start studio54
sudo systemctl stop studio54
sudo systemctl enable studio54   # enable auto-start
sudo systemctl disable studio54  # disable auto-start
```

## MusicBrainz Local Mirror (Optional)

Eliminates the 1 req/sec MusicBrainz API rate limit. Requires ~70 GB disk space and a free MetaBrainz account.

```bash
./studio54 musicbrainzdb setup <your-replication-token>
```

Get a token at: https://metabrainz.org/supporters/account-type

## Data Storage

All data is stored on the host (not in Docker volumes) for easy backup:

```
/docker/studio54/
├── postgres/        # Database files
├── redis/           # Redis persistence
└── cover-art/       # Album and artist images
```

Backup and restore:

```bash
./studio54 backup all
./studio54 restore data/backups/studio54-db_20260101_120000.tar.gz
```

## Watchtower (Auto-Updates)

Studio54 includes optional Watchtower integration for automatic container updates:

```bash
# Enable auto-updates
./studio54 watchtower enable

# Control per-service updates in .env:
WATCHTOWER_STUDIO54_SERVICE=true
WATCHTOWER_STUDIO54_DB=false      # never auto-update the database
WATCHTOWER_INTERVAL=86400         # check daily (seconds)
```

## Port Reference

If defaults conflict with other services, change them in `.env`:

```bash
STUDIO54_WEB_PORT=8009
STUDIO54_SERVICE_PORT=8010
STUDIO54_DB_PORT=5434
STUDIO54_REDIS_PORT=6381
STUDIO54_DOZZLE_PORT=9998
```

## Troubleshooting

```bash
# Check prerequisites
./studio54 check

# View logs
./studio54 logs
./studio54 logs studio54-service
./studio54 logs studio54-worker

# Check container status
docker ps --filter "name=studio54"
```

**Port conflicts:**
```bash
sudo lsof -i :8009   # Web UI
sudo lsof -i :8010   # API
sudo lsof -i :5434   # Database
```

**Database won't start:** Check that `STUDIO54_DATA_DIR` exists and is writable, and that `STUDIO54_DB_PASSWORD` is set in `.env`.

**Worker not processing tasks:** Check Redis is healthy (`docker ps | grep studio54-redis`) and the worker logs (`./studio54 logs studio54-worker`).
