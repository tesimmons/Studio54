# Studio54 ‚Äî iOS App Plan (Capacitor)

**Status:** On Hold ‚Äî ready to resume  
**Distribution:** App Store  
**Backend:** https://studio54.homeip.net  
**Approach:** Capacitor wraps the existing React/Vite web app in a native iOS shell

---

## Overview

Capacitor embeds the built Studio54 web app into a native WKWebView. The app runs on the phone exactly as it does in a browser, but ships as a proper `.ipa` and goes through App Store review. Native plugins fill the gaps the WebView cannot handle (background audio, lock-screen controls).

## Phase 1 ‚Äî Capacitor Scaffold

1. Add Capacitor to `studio54-web`:
   ```
   npm install @capacitor/core @capacitor/cli @capacitor/ios
   npx cap init "Studio54" "net.homeip.studio54"
   ```
2. Set `webDir = dist` in `capacitor.config.ts`
3. Set `server.url = https://studio54.homeip.net` for live-reload dev; remove for production build so app ships bundled HTML
4. Add `VITE_API_URL=https://studio54.homeip.net/api/v1` to `.env.production`
5. `npm run build && npx cap add ios && npx cap sync`
6. Open `ios/App/App.xcworkspace` in Xcode ‚Äî verify it runs in Simulator

## Phase 2 ‚Äî Background Audio (Critical)

iOS kills audio playback when the app backgrounds unless the app declares the `audio` background mode.

**Plugin:** `@capacitor-community/background-mode` or native Xcode capability toggle  
**Lock-screen / Now Playing widget:** requires `MPNowPlayingInfoCenter` and `MPRemoteCommandCenter` ‚Äî use `capacitor-plugin-music-controls` or write a thin native Swift plugin

Tasks:
- Enable **Audio, AirPlay, and Picture in Picture** background capability in Xcode
- Wire `MPNowPlayingInfoCenter` to update when `currentTrack` changes in the React app (Capacitor bridge call)
- Handle lock-screen transport controls (play/pause/next/prev) firing back into the React player

## Phase 4 ‚Äî App Store Submission

1. Register **App ID** `net.homeip.studio54` in developer.apple.com
2. Create **Distribution Certificate** + **Provisioning Profile**
3. Set version / build number in Xcode (`MARKETING_VERSION`, `CURRENT_PROJECT_VERSION`)
4. Archive ‚Üí Distribute ‚Üí App Store Connect
5. Fill out App Store Connect listing:
   - App name, subtitle, description, keywords
   - Screenshots (6.9" iPhone required, 6.5" optional)
   - Privacy policy URL (required ‚Äî must host one at homeip.net)
   - Age rating, category (Music)
6. Submit for review (typical: 1‚Äì3 days)
